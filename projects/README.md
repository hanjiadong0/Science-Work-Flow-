Recommended additional link:

- add more study repos here if you want other course work linked into the same workflow

Why this helps:

- the full data science root gives you a top-level entry point across courses and semesters
- the parent roots keep course-level and topic-level context visible
- your study repo stays the place for notes, practice, and method summaries
- this repo stays the place for structured experiments, notebooks, and evaluation runs
- both repos become easy to use together from one workspace layout

The junction is intentionally ignored by git so the large external Lean project does not get committed into this repository.
